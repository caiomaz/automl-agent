## Phase 3: Feature Engineering

### Step 3.1 — Categorical Encoding
- **Low-cardinality categoricals** (e.g., `Sex` with ≤ 10 unique values): Use **one-hot encoding** (drop first to avoid collinearity).
- **High-cardinality categoricals** (if any, > 10 unique values): Use **target encoding** with leave-one-out strategy to prevent leakage, fitted only on the training set.

### Step 3.2 — Numerical Feature Transformations
- Apply **Yeo-Johnson power transformation** (via `PowerTransformer(method='yeo-johnson')`) to numerical features exhibiting high skewness (|skew| > 0.75). Fit on training set only.
- **Standard scaling** (zero mean, unit variance) on all numerical features after transformation. Fit on training set only.

### Step 3.3 — Derived Features (Domain-Informed for Biology)
Given the crab age prediction context, engineer the following biologically plausible features:
- **Volume proxy**: `Length × Diameter × Height` (approximates shell volume).
- **Density proxy**: `Weight / (Length × Diameter × Height + 1e-8)` (shell density indicator).
- **Weight ratios** (if multiple weight columns exist, e.g., Shucked Weight, Viscera Weight, Shell Weight):
  - `Shell_Weight_Ratio = Shell_Weight / Weight`
  - `Meat_Weight_Ratio = Shucked_Weight / Weight`
  - `Viscera_Weight_Ratio = Viscera_Weight / Weight`
- **Size-to-weight ratio**: `Length / (Weight + 1e-8)`.
- These derived features should be computed **before** scaling but **after** missing value imputation.

### Step 3.4 — Feature Selection
- **Variance threshold**: Remove features with near-zero variance (variance < 1e-5).
- **Correlation filter**: Among feature pairs with |r| > 0.95, drop the feature with lower correlation to the target.
- **Univariate importance**: Compute mutual information between each feature and the target; retain all features (informational, no aggressive pruning at this stage).

## Phase 3: Data Preprocessing & Feature Engineering

5. **Constant columns**: Drop any zero-variance features.
6. **Negative age values**: Drop rows where `age < 0`.

### Step 3.2 — Feature Engineering

Create the following derived features (if the base columns exist in the dataset):

1. **Volume-like feature**: If length, diameter, and height columns exist, compute `volume_proxy = length × diameter × height`.

   - `shell_weight_ratio = shell_weight / (whole_weight + 1e-8)`

   - `viscera_weight_ratio = viscera_weight / (whole_weight + 1e-8)`
   - `meat_to_shell_ratio = shucked_weight / (shell_weight + 1e-8)`
3. **Size-to-weight ratios**: If length/diameter/height and weight columns exist:
   - `length_to_weight = length / (whole_weight + 1e-8)`
   - `height_to_length = height / (length + 1e-8)`
4. **Log transforms**: For any heavily skewed numerical feature (skewness > 1.0), create a log1p-transformed version: `feature_log = np.log1p(feature)`.

### Step 3.3 — Encode Categorical Features
