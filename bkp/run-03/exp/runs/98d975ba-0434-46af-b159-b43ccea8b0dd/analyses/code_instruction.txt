## 3. Model Architecture & Hyperparameters
Select **LightGBM Classifier** as the single model solution. This model offers the best balance of speed (approx. 12s training) and accuracy (F1-Score ~0.92) for this specific constraint.

### 3.1 Framework Note
*   **PyTorch Preference:** While PyTorch is preferred for deep learning, it is **not** used here. LightGBM is a gradient boosting tree algorithm, not a neural network. Using PyTorch for a simple MLP would risk exceeding the 5-minute training limit and is less efficient for tabular data.
*   **Implementation:** Use the `lightgbm` library directly.

### 3.2 Hyperparameter Configuration
Use the following specific hyperparameters derived from the MLE team's Optuna analysis. Do not deviate unless Optuna explicitly overrides them during the 20-trial search.

    *   `n_estimators`: **100**
    *   `feature_fraction`: **0.8**
    *   `class_weight`: **'balanced'**
*   **Optimization:**
    *   **Engine:** Optuna.
    *   **Trials:** Maximum **20 trials**.
    *   **Objective:** Maximize F1-Score on the **Validation Split**.
    *   **Time Budget:** Hard stop at **3 minutes** for HPO to ensure total pipeline time < 5 minutes.
